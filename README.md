検査前処置アプリ（フロントエンド）
概要

内視鏡検査前処置の進捗を
患者入力 と スタッフ確認 に分けて管理するフロントエンドです。
本リポジトリは フロント専用（HTML / CSS / JavaScript） です。

画面構成
患者画面（index.html）

患者が前処置の進捗を入力する画面。
・💊 下剤
・💩 排便（便性状画像選択）
・⚠️ 症状（腹痛・吐き気・その他）
・入力時刻は自動取得
・患者側では一覧表示なし

スタッフ画面（staff.html）
・検査進捗を確認する一覧画面。
・検査ID / 検査時間 / 薬の種類を表示
・検査記録一覧を表示（下剤・排便・症状）

取消（訂正）仕様
方針
データは物理削除しない（医療用途想定）

動作
・🗑 クリック → 確認モーダル（〇 / ×）
・〇を押すと：
　行は削除しない
　行全体を 薄いグレー表示（訂正済み）
　回数表示（例：下剤(3)）は変更しない

※ 将来DBでは「訂正フラグ」で管理予定

技術構成
Kensa_frontend/
├ index.html   # 患者画面
├ staff.html   # スタッフ画面
├ style.css    # 共通スタイル
└ script.js    # 操作・表示制御

JavaScriptの役割（script.js）
・記録追加（下剤 / 排便 / 症状）
・時刻取得
・一覧行生成
・取消（訂正）モーダル制御
・行番号（No）付与
・スタッフ画面用ダミーデータ表示
